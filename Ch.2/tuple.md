# 튜플 자료형 (Tuples)

---
튜플은 리스트와 거의 비슷하지만, 가장 결정적인 차이점은 **한 번 생성하면 그 값을 절대 바꿀 수 없다(Immutable)**는 점입니다.

## **📦 튜플 만들기**
<br>
튜플은 소괄호 ( )를 사용하여 정의하며, 요소를 쉼표(,)로 구분합니다.

### **🚀 TRY IT!**
{: .text-blue-200 }

```python
t1 = (1, 2, 3)
t2 = 1, 2, 3          # 소괄호 생략 가능
t3 = ('apple', 3)     # 서로 다른 자료형(이질적 데이터)을 담기에 적합
t4 = (1,)             # 요소가 하나일 때는 반드시 뒤에 쉼표(,)를 붙여야 함!
```

### **❓ 리스트가 있는데 튜플은 왜 쓰나요?**

|**특징**|**리스트 (Lists)**|**튜플 (Tuples)**|
|:-------|:-------|:-------|
|가변성|Mutable (수정/삭제 가능) |Immutable (수정/삭제 불가능)|
|데이터 성격|보통 같은 종류의 데이터 목록|서로 연관된 다른 종류의 데이터 쌍|
|예시|`['apple', 'banana', 'pear']`| `('apple', 3)` (사과가 3개 있다는 뜻)|


{: .highlight }
**데이터의 안전성**:  
프로그램이 실행되는 동안 값이 변하지 않아야 하는 중요한 정보(예: 좌표, 설정값)는 튜플에 담는 것이 안전합니다. 실수로 값을 바꾸려 하면 파이썬이 즉시 에러를 내주기 때문입니다.

## **튜플의 금기 사항 (Immutability)**
튜플은 요소를 수정하거나 삭제하려고 하면 TypeError가 발생합니다.

### **🚀 TRY IT!**
{: .text-blue-200 }

```python
t = (1, 2, 3)

# 아래 코드는 모두 에러가 발생합니다!
del t[0]     # 삭제 불가
t[0] = 10    # 수정 불가
```

---

## **튜플 다루기 (기본 연산)**
값을 바꿀 수 없다는 점만 제외하면 인덱싱, 슬라이싱, 연산 방식은 리스트와 똑같습니다.

### **🚀 TRY IT!**
{: .text-blue-200 }

```python
t = (1, 2, 3, 2)

print(len(t))        # 길이: 4
print(t[1:3])        # 슬라이싱: (2, 3)
print(t.count(2))    # 요소 개수 세기: 2
```

## **패킹(Packing) & 언패킹(Unpacking)**
여러 값을 하나의 튜플로 묶거나, 튜플 안의 값을 여러 변수에 한 번에 할당할 수 있습니다.

### **🚀 TRY IT!**
{: .text-blue-200 }

```python
# 패킹 (Packing)
fruits = ("apple", "banana", "cherry")

# 언패킹 (Unpacking)
a, b, c = fruits
print(a) # 'apple'
print(b) # 'banana'
```

### **🤔 Wait and Think!**
❓ 왜 t = (1)은 튜플이 아닐까요?
파이썬은 소괄호 ( )를 수학 연산의 우선순위 기호로도 사용합니다. 그래서 요소가 하나인 튜플을 만들 때는 **(1,)**처럼 쉼표를 찍어 "이건 연산 기호가 아니라 튜플이야!"라고 알려줘야 합니다.
