# 반복문2 (While Loops)
<br>
`while` 문은 주어진 논리 표현식(Logical Expression)이 참(True)인 동안 코드 블록을 무한히 반복합니다. 반복 횟수가 미리 정해져 있지 않은 경우가 많아 **부정적 반복문(Indefinite loop)**이라고도 불립니다.

## **🛠️ 기본 문법**

```python
while 조건문:
    수행할 문장 1
    수행할 문장 2
```

- **`while` 키워드**: 반복의 시작을 알립니다.
- **조건문**: 반복을 계속할지 결정하는 논리 표현식입니다. 이 식의 결과가 `True`이면 반복하고, `False`가 되면 즉시 멈춥니다.
- **콜론(:)과 들여쓰기**: `while` 문에 속한 실행문들은 반드시 동일한 간격으로 들여쓰기 되어야 합니다. 들여쓰기가 끝나는 지점이 반복문이 끝나는 지점입니다.

**🔄 동작 원리**

1. 먼저 `while` 뒤의 조건문이 참인지 거짓인지 검사합니다.
2. 조건이 **참(`True`)**이라면 들여쓰기 된 코드 블록을 실행합니다.
3. 록 내의 모든 문장을 실행한 후, 다시 위로 돌아가 **1번(조건 판단)**부터 반복합니다.
4. 조건이 **거짓(`False`)**이 되면 반복문을 빠져나와 다음 코드로 진행합니다.

---

### **🚀 TRY IT!**
{: .text-blue-200 }

```python
# 1부터 3까지 출력하기
count = 1

while count <= 3:
    print(f"현재 숫자는 {count}입니다.")
    count = count + 1  # 이 문장이 없으면 무한 루프에 빠집니다!

print("반복이 끝났습니다.")
```
> {: .result .fs-3 }  
> 현재 숫자는 1입니다.  
> 현재 숫자는 2입니다.  
> 현재 숫자는 3입니다.  
> 반복이 끝났습니다.  

### **🚀 TRY IT!**
{: .text-blue-200 }

숫자 8을 2로 계속 나누어, 그 결과가 1 미만이 될 때까지 몇 번이나 나눌 수 있는지 알아봅시다.

```python
i = 0
n = 8

while n >= 1:
    n /= 2  # n을 2로 나눔
    i += 1  # 나눈 횟수를 1 증가
    print(f'n = {n}, i = {i}')
```
> {: .result .fs-3 }  
> n = 4.0, i = 1  
> n = 2.0, i = 2  
> n = 1.0, i = 3  
> n = 0.5, i = 4  

### **🤔 Wait and Think!**
위의 예시 코드를 실행할 때 어떤 일이 일어나고 있는 것일까요?

1. 초기 설정: 변수 i(나눈 횟수)는 0으로, n(나눌 숫자)은 8로 설정됩니다.

2. 반복 1회차
- 조건 검사: n >= 1 (8 >= 1)은 **참(True)**이므로 코드를 실행합니다.
- n은 8 / 2 = 4.0이 되고, i는 1이 됩니다.

3. 반복 2회차
- 조건 검사: n >= 1 (4 >= 1)은 **참(True)**이므로 코드를 실행합니다.
- n은 4 / 2 = 2.0이 되고, i는 2이 됩니다.

4. 반복 3회차
- 조건 검사: n >= 1 (2 >= 1)은 **참(True)**이므로 코드를 실행합니다.
- n은 2 / 2 = 1.0이 되고, i는 3이 됩니다.

5. 반복 4회차
- 조건 검사: n >= 1 (1 >= 1)은 **참(True)**이므로 코드를 실행합니다.
- n은 1 / 2 = 0.5가 되고, i는 4가 됩니다.

6. 반복 종료
- 조건 검사: n >= 1 (0.5 >= 1)은 **거짓(False)**입니다.
- 조건이 만족되지 않으므로 while 문을 종료합니다. 최종적인 i의 값은 4가 됩니다.

## **⚠️ 무한 루프 (Infinite Loops)**

조건문이 항상 `True`여서 결코 끝나지 않는 반복을 의미합니다.

```python
while True:
    print("Press ctrl+c if you want to quit the while-loop")
```

- 발생 원인: 루프 내부에서 조건을 `False`로 바꿀 장치가 없거나, 조건 설계가 잘못된 경우입니다.
- 탈출 방법: 터미널이나 에디터에서 `ctrl+c`를 눌러 강제로 종료합니다.
- 실전 활용: 서버 통신 대기, 실시간 입력 감지 등 언제 끝날지 모르는 상황에서는 의도적으로 `while True:`를 사용하기도 합니다.

## **🛠️ 루프 제어문 (break, continue)**

반복문이 실행되는 도중, 특정 조건에 따라 루프를 완전히 중단하거나 일부를 건너뛰어야 할 때 사용합니다.

### **🚫 break: "즉시 가동 중단!"**
`break` 키워드는 조건문의 참/거짓 여부와 상관없이 자신을 감싸고 있는 가장 가까운 반복문을 **즉시 종료**하고 빠져나옵니다.

🤔 언제 쓸까요? 무한 루프에서 특정 목표치에 도달했거나, 에러가 발생하여 더 이상 계산이 불가능할 때 사용합니다.

```python
pressure = 80  # 초기 압력

while True:  # 무한 루프 가동
    print(f"현재 압력: {pressure}")
    
    if pressure >= 100:
        print("⚠️ 위험! 임계 압력 도달. 시스템을 비상 정지합니다.")
        break  # 루프를 즉시 탈출!
    
    pressure += 5  # 압력이 서서히 상승한다고 가정
```

### **⏭️ continue: "이번 판은 건너뛰자!"**
`continue` 문은 반복문을 완전히 빠져나가는 것이 아니라, 현재 차례의 남은 코드를 실행하지 않고 다시 `while` 문의 조건 검사(처음)로 돌아가게 합니다.

🤔 언제 쓸까요? 센서 오작동으로 인한 음수 데이터나 결측치처럼 분석에 필요 없는 데이터를 제외하고 계산을 이어가고 싶을 때 유용합니다.

```python
sensor_data = [15.5, 20.1, -999, 18.3, -1, 22.4] # -999와 -1은 오류값
index = -1

while index < len(sensor_data) - 1:
    index += 1
    value = sensor_data[index]
    
    if value < 0:
        continue  # 음수 데이터라면 아래 print를 실행하지 않고 다시 처음으로!
        
    print(f"정상 데이터 수집: {value}")
```

## **중첩된 While 문**

`if`문 `for`문과 마찬가지로 `while`문 안에 또 다른 `while`문을 사용할 수 있다. 이를 중첩된 `while` 문이라고 한다. 

중첩된 `for`문과 `while`문을 익히는데 구구단만한 것이 없다. 지난 시간에 `for`문으로 연습했던 구구단을 `while`문을 사용해서 구현할 수 있다.

```python
>>> i = 2
>>> while i <= 3:
        j = 1
        while j <= 9:
            print(f"{i} x {j} = {i*j}")
            j += 1
        i += 1
```

{: .warning }
> 중첩된 while 문에서 break나 continue를 사용하면 가장 가까운 while 문에만 영향을 줍니다.
