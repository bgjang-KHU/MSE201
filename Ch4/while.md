# 반복문2 (While Loops)
<br>
`while` 문은 주어진 논리 표현식(Logical Expression)이 참(True)인 동안 코드 블록을 무한히 반복합니다. 반복 횟수가 미리 정해져 있지 않은 경우가 많아 **부정적 반복문(Indefinite loop)**이라고도 불립니다.

## **🛠️ 기본 문법**

```python
while 조건문:
    수행할 문장 1
    수행할 문장 2
```

- **`while` 키워드**: 반복의 시작을 알립니다.
- **조건문**: 반복을 계속할지 결정하는 논리 표현식입니다. 이 식의 결과가 `True`이면 반복하고, `False`가 되면 즉시 멈춥니다.
- **콜론(:)과 들여쓰기**: `while` 문에 속한 실행문들은 반드시 동일한 간격으로 들여쓰기 되어야 합니다. 들여쓰기가 끝나는 지점이 반복문이 끝나는 지점입니다.

**🔄 동작 원리**

1. 먼저 `while` 뒤의 조건문이 참인지 거짓인지 검사합니다.
2. 조건이 **참(`True`)**이라면 들여쓰기 된 코드 블록을 실행합니다.
3. 록 내의 모든 문장을 실행한 후, 다시 위로 돌아가 **1번(조건 판단)**부터 반복합니다.
4. 조건이 **거짓(`False`)**이 되면 반복문을 빠져나와 다음 코드로 진행합니다.

---

### **🚀 TRY IT!**
{: .text-blue-200 }

```python
# 1부터 3까지 출력하기
count = 1

while count <= 3:
    print(f"현재 숫자는 {count}입니다.")
    count = count + 1  # 이 문장이 없으면 무한 루프에 빠집니다!

print("반복이 끝났습니다.")
```
> {: .result .fs-3 }  
> 현재 숫자는 1입니다.  
> 현재 숫자는 2입니다.  
> 현재 숫자는 3입니다.  
> 반복이 끝났습니다.  

### **🚀 TRY IT!**
{: .text-blue-200 }

숫자 8을 2로 계속 나누어, 그 결과가 1 미만이 될 때까지 몇 번이나 나눌 수 있는지 알아봅시다.

```python
i = 0
n = 8

while n >= 1:
    n /= 2  # n을 2로 나눔
    i += 1  # 나눈 횟수를 1 증가
    print(f'n = {n}, i = {i}')
```
> {: .result .fs-3 }  
> n = 4.0, i = 1  
> n = 2.0, i = 2  
> n = 1.0, i = 3  
> n = 0.5, i = 4  

### **🤔 Wait and Think!**
위의 예시 코드를 실행할 때 어떤 일이 일어나고 있는 것일까요?

1. 초기 설정: 변수 i(나눈 횟수)는 0으로, n(나눌 숫자)은 8로 설정됩니다.

2. 반복 1회차
- 조건 검사: n >= 1 (8 >= 1)은 **참(True)**이므로 코드를 실행합니다.
- n은 8 / 2 = 4.0이 되고, i는 1이 됩니다.

3. 반복 2회차
- 조건 검사: n >= 1 (4 >= 1)은 **참(True)**이므로 코드를 실행합니다.
- n은 4 / 2 = 2.0이 되고, i는 2이 됩니다.

4. 반복 3회차
- 조건 검사: n >= 1 (2 >= 1)은 **참(True)**이므로 코드를 실행합니다.
- n은 2 / 2 = 1.0이 되고, i는 3이 됩니다.

5. 반복 4회차
- 조건 검사: n >= 1 (1 >= 1)은 **참(True)**이므로 코드를 실행합니다.
- n은 1 / 2 = 0.5가 되고, i는 4가 됩니다.

6. 반복 종료
- 조건 검사: n >= 1 (0.5 >= 1)은 **거짓(False)**입니다.
- 조건이 만족되지 않으므로 while 문을 종료합니다. 최종적인 i의 값은 4가 됩니다.

### **🚀 TRY IT!**
{: .text-blue-200 }

```python
while True:
    print("Press ctrl+c if you want to quit the while-loop")
```

**⚠️ 무한 루프 (Infinite Loops)**
조건문이 항상 `True`여서 결코 끝나지 않는 반복을 의미합니다.

- 발생 원인: 루프 내부에서 조건을 `False`로 바꿀 장치가 없거나, 조건 설계가 잘못된 경우입니다.
- 탈출 방법: 터미널이나 에디터에서 `ctrl+c`를 눌러 강제로 종료합니다.
- 실전 활용: 서버 통신 대기, 실시간 입력 감지 등 언제 끝날지 모르는 상황에서는 의도적으로 `while True:`를 사용하기도 합니다.
